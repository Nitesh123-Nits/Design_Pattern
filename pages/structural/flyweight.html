Flyweight Pattern

Uses sharing to efficiently support a large number of fine-grained objects. Splits object state into intrinsic (shared, immutable) and extrinsic (unique, passed in).

When to Use

  • When you have a massive number of similar objects consuming too much memory
  • Character rendering in text editors (millions of chars, share font/style data)
  • Particle systems in games
  • Integer, String caching in Java (Integer.valueOf() pool)

Java Implementation

Text Editor Character Flyweight
import java.util.HashMap;
import java.util.Map;

// Flyweight — contains intrinsic (shared) state
public final class CharacterStyle {
    private final String fontFamily;  // intrinsic: shared
    private final int fontSize;        // intrinsic: shared
    private final String color;        // intrinsic: shared
    private final boolean bold;        // intrinsic: shared

    public CharacterStyle(String fontFamily, int fontSize, String color, boolean bold) {
        this.fontFamily = fontFamily;
        this.fontSize   = fontSize;
        this.color      = color;
        this.bold       = bold;
    }

    // Render with extrinsic state passed in
    public void render(char character, int x, int y) {
        System.out.printf("Char '%c' at (%d,%d) | %s %dpx %s %s%n",
            character, x, y, fontFamily, fontSize, color, bold ? "BOLD" : "");
    }

    @Override
    public String toString() { return fontFamily + "-" + fontSize + "-" + color + "-" + bold; }
}

// Flyweight Factory — creates and caches flyweights
public class CharacterStyleFactory {
    private static final Map<String, CharacterStyle> cache = new HashMap<>();

    public static CharacterStyle getStyle(String fontFamily, int fontSize, String color, boolean bold) {
        String key = fontFamily + "-" + fontSize + "-" + color + "-" + bold;
        return cache.computeIfAbsent(key,
            k -> {
                System.out.println("Creating NEW style: " + k);
                return new CharacterStyle(fontFamily, fontSize, color, bold);
            }
        );
    }

    public static int getCacheSize() { return cache.size(); }
}

// Context — holds extrinsic state (unique per character instance)
public class DocumentCharacter {
    private final char character;    // extrinsic: unique per char
    private final int x, y;          // extrinsic: position
    private final CharacterStyle style; // shared flyweight

    public DocumentCharacter(char character, int x, int y, CharacterStyle style) {
        this.character = character;
        this.x = x;
        this.y = y;
        this.style = style;
    }

    public void render() {
        style.render(character, x, y); // pass extrinsic state to flyweight
    }
}

// Usage — 1 million chars, but only a handful of style objects
public class TextDocument {
    public static void main(String[] args) {
        // Imagine rendering a large document
        String text = "Hello World Java Design Patterns";
        int x = 0;

        for (int i = 0; i < text.length(); i++) {
            // All chars share the SAME style object — not millions of style objects
            CharacterStyle style = CharacterStyleFactory.getStyle("Arial", 12, "black", false);
            DocumentCharacter ch = new DocumentCharacter(text.charAt(i), x, 100, style);
            ch.render();
            x += 8;
        }

        // Bold title — different but reused style
        CharacterStyle titleStyle = CharacterStyleFactory.getStyle("Arial", 24, "blue", true);
        new DocumentCharacter('J', 0, 0, titleStyle).render();

        // Only 2 style objects created for entire document!
        System.out.println("\nTotal style objects: " + CharacterStyleFactory.getCacheSize()); // 2

        // Real Java Flyweight examples:
        Integer a = Integer.valueOf(127);  // cached
        Integer b = Integer.valueOf(127);  // same object!
        System.out.println("Same integer object? " + (a == b)); // true (-128 to 127 cached)
    }
}

Interview Q&A

Intrinsic state: Stored in the flyweight, shared across all instances, independent of context. Should be immutable. Example: font family, font size — same for all 'A' characters in the same style.
Extrinsic state: Unique to each instance, cannot be shared, passed in at runtime by the client. Example: x/y position of each character, the actual character value.

Java's String interning pool is a Flyweight implementation. String literals in Java are automatically interned — "hello" == "hello" is true because both point to the same String object in the pool. String.intern() explicitly pools a string. Integer.valueOf() caches integers between -128 and 127 for the same reason.
