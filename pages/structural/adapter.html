6. Adapter Pattern

Converts the interface of a class into another interface that clients expect. The Adapter pattern lets classes work together that couldn't otherwise because of incompatible interfaces.

When to Use

  • When you want to use an existing class, but its interface doesn't match what you need
  • When integrating third-party libraries
  • When working with legacy code that cannot be modified
  • Real-world: InputStreamReader (bytes → characters), JDBC adapters, Spring's HandlerAdapter

public class StripePaymentSDK {
    public String chargeCard(double amountInCents, String curr, Map<String, String> metadata) {

        return "stripe_txn_" + System.currentTimeMillis();
    }

    public Map<String, Object> retrieveCharge(String chargeId) {
        return Map.of("id", chargeId, "status", "succeeded", "amount", 9999);

Payment Gateway Adapter - Adapter 1 — Stripe
//  Adapter 1 — Stripe 
public class StripeAdapter implements PaymentGateway {
    private final StripePaymentSDK stripe;

    public StripeAdapter(StripePaymentSDK stripe) {
        this.stripe = stripe;
    }

    @Override
    public boolean processPayment(String orderId, double amount, String currency) {
        // Adapt: convert dollar amount to cents, wrap in metadata
        double amountInCents = amount * 100;
        Map<String, String> metadata = Map.of("order_id", orderId);
        String chargeId = stripe.chargeCard(amountInCents, currency, metadata);
        System.out.println("Adapter mapped Stripe charge ID: " + chargeId);
        return chargeId != null;
    }

    @Override
    public String getTransactionStatus(String transactionId) {
        Map<String, Object> charge = stripe.retrieveCharge(transactionId);
        return (String) charge.get("status"); // adapt response format
    }
}
Payment Gateway Adapter - Adapter 2 — PayPal
//  Adapter 2 — PayPal 
public class PayPalAdapter implements PaymentGateway {
    private final PayPalLegacySDK paypal;
    private String lastOrderId;

    public PayPalAdapter(PayPalLegacySDK paypal) { this.paypal = paypal; }

    @Override
    public boolean processPayment(String orderId, double amount, String currency) {
        this.lastOrderId = orderId;
        paypal.initiatePayment(orderId, (float) amount); // adapt double to float
        return true;
    }

    @Override
    public String getTransactionStatus(String transactionId) {
        return paypal.checkPaymentState(lastOrderId); // adapt method name
    }
}
Payment Gateway Adapter - Client — only knows PaymentGateway interface
//  Client — only knows PaymentGateway interface 
public class OrderCheckoutService {
    private final PaymentGateway gateway;

    public OrderCheckoutService(PaymentGateway gateway) {
        this.gateway = gateway; // inject adapter
    }

    public void checkout(String orderId, double amount) {
        boolean success = gateway.processPayment(orderId, amount, "USD");
        System.out.println("Payment " + (success ? "successful" : "failed"));
    }
}
Payment Gateway Adapter - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        // Use Stripe — client totally unaware of Stripe's API
        PaymentGateway stripeGateway = new StripeAdapter(new StripePaymentSDK());
        new OrderCheckoutService(stripeGateway).checkout("ORD-001", 99.99);

        // Switch to PayPal — same client code, different adapter
        PaymentGateway paypalGateway = new PayPalAdapter(new PayPalLegacySDK());
        new OrderCheckoutService(paypalGateway).checkout("ORD-002", 49.99);
    }
}

Real-World Usage in Java

  • java.io.InputStreamReader — adapts InputStream (bytes) to Reader (chars)
  • java.io.OutputStreamWriter — adapts OutputStream to Writer
  • java.util.Arrays.asList() — adapts arrays to List
  • Spring's HandlerAdapter — adapts different controller types to MVC framework
  • Spring's JpaVendorAdapter — adapts different JPA implementations

Interview Q&A

Adapter: Changes the interface of an object. Makes incompatible interfaces compatible. "I need this to look like that."
Decorator: Keeps the same interface but adds new behavior/responsibilities. "I want this to do more while looking the same." Both wrap an object, but their intent differs.

Object Adapter (preferred in Java): Adapter uses composition — holds a reference to the adaptee. More flexible, can adapt multiple related classes.
Class Adapter: Adapter inherits from the adaptee. Not possible in Java for classes (only one class inheritance), but achievable with interfaces. Less flexible since static binding to one adaptee class.

Adapter: Applied after the fact — fixes incompatible interfaces that weren't designed to work together. Usually used for integration/legacy code.
Bridge: Applied proactively at design time — intentionally separates abstraction from implementation so both can vary independently. Bridge is about design; Adapter is about integration.
