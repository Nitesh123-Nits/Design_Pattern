🌳 Composite Pattern

Composes objects into tree structures to represent part-whole hierarchies. Lets clients treat individual objects and compositions of objects uniformly.

🎯 When to Use

  • File system (files and folders)
  • Org chart hierarchies
  • UI component trees (panels containing sub-panels and widgets)
  • Expression trees (parsers, compilers)

💻 Java Implementation

File System — Classic Composite
import java.util.ArrayList;
import java.util.List;

// Component — unified interface for leaves and composites
public abstract class FileSystemItem {
    protected final String name;
    public FileSystemItem(String name) { this.name = name; }
    public abstract long getSize();
    public abstract void printStructure(String indent);
    // Optional: add/remove — only composites implement these meaningfully
    public void add(FileSystemItem item) { throw new UnsupportedOperationException(); }
    public void remove(FileSystemItem item) { throw new UnsupportedOperationException(); }
}

// Leaf — no children
public class File extends FileSystemItem {
    private final long sizeBytes;

    public File(String name, long sizeBytes) {
        super(name);
        this.sizeBytes = sizeBytes;
    }

    @Override public long getSize() { return sizeBytes; }

    @Override
    public void printStructure(String indent) {
        System.out.printf("%s📄 %s (%d KB)%n", indent, name, sizeBytes / 1024);
    }
}

// Composite — has children (can be files or other folders)
public class Folder extends FileSystemItem {
    private final List<FileSystemItem> children = new ArrayList<>();

    public Folder(String name) { super(name); }

    @Override public void add(FileSystemItem item)    { children.add(item); }
    @Override public void remove(FileSystemItem item) { children.remove(item); }

    @Override
    public long getSize() {
        // Recursive — folder size = sum of all children sizes
        return children.stream().mapToLong(FileSystemItem::getSize).sum();
    }

    @Override
    public void printStructure(String indent) {
        System.out.printf("%s📁 %s/ (%d KB total)%n", indent, name, getSize() / 1024);
        children.forEach(child -> child.printStructure(indent + "  "));
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        // Build file system tree
        Folder root = new Folder("root");

        Folder src = new Folder("src");
        src.add(new File("Main.java", 5120));
        src.add(new File("App.java", 3072));

        Folder resources = new Folder("resources");
        resources.add(new File("application.properties", 1024));
        resources.add(new File("schema.sql", 2048));

        Folder docs = new Folder("docs");
        docs.add(new File("README.md", 8192));
        docs.add(new File("CHANGELOG.md", 4096));

        root.add(src);
        root.add(resources);
        root.add(docs);
        root.add(new File("pom.xml", 6144));

        // Client treats files and folders uniformly!
        root.printStructure("");
        System.out.println("\nTotal size: " + root.getSize() / 1024 + " KB");
        System.out.println("Docs size: " + docs.getSize() / 1024 + " KB");
    }
}

🎤 Interview Q&A

Clients can treat individual objects (leaves) and compositions (composites) with the same interface. You don't need instanceof checks to determine if you're dealing with a file or folder. Recursive operations over the tree are elegant and clean. Adding new leaf types doesn't require changes to existing code.

(1) Spring Security's AuthenticationManager — ProviderManager is a composite that delegates to multiple AuthenticationProviders. (2) Spring's CompositePasswordEncoder. (3) Spring MVC's HandlerInterceptorAdapter chains. (4) Spring's CompositeMessageSource for i18n.
