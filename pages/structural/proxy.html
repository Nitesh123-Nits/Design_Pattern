🔒 Proxy Pattern

Provides a surrogate or placeholder for another object. A proxy controls access to the original object, allowing you to perform something before or after the request reaches the original object.

🎯 Types of Proxy

  • Virtual Proxy — Lazy initialization; create expensive objects only when needed
  • Protection Proxy — Controls access based on permissions
  • Remote Proxy — Represents object in different address space (RMI, REST client)
  • Caching Proxy — Caches results of expensive operations
  • Logging Proxy — Adds logging before/after method calls

💻 Java Implementation

Caching + Protection Proxy
import java.util.HashMap;
import java.util.Map;

// Subject Interface
public interface DatabaseService {
    String query(String sql);
    void execute(String sql);
}

// Real Subject — expensive operations
public class RealDatabaseService implements DatabaseService {
    public RealDatabaseService() {
        System.out.println("🔌 Connecting to database...");
    }

    @Override
    public String query(String sql) {
        System.out.println("⚡ DB QUERY: " + sql);
        return "Result for: " + sql;
    }

    @Override
    public void execute(String sql) {
        System.out.println("⚡ DB EXECUTE: " + sql);
    }
}

// ── Proxy 1: Caching Proxy ────────────────────────────────────
public class CachingDatabaseProxy implements DatabaseService {
    private DatabaseService realService;  // lazy — created only when needed
    private final Map<String, String> cache = new HashMap<>();

    private DatabaseService getRealService() {
        if (realService == null) {
            realService = new RealDatabaseService();  // Virtual Proxy: lazy init
        }
        return realService;
    }

    @Override
    public String query(String sql) {
        // Check cache first
        if (cache.containsKey(sql)) {
            System.out.println("✅ CACHE HIT: " + sql);
            return cache.get(sql);
        }
        // Cache miss — delegate to real service
        String result = getRealService().query(sql);
        cache.put(sql, result);
        System.out.println("💾 CACHED result for: " + sql);
        return result;
    }

    @Override
    public void execute(String sql) {
        // No caching for writes — invalidate cache
        cache.clear();
        getRealService().execute(sql);
    }
}

// ── Proxy 2: Protection Proxy (Role-Based Access) ──────────────
public class SecureDatabaseProxy implements DatabaseService {
    private final DatabaseService delegate;
    private final String userRole;

    public SecureDatabaseProxy(DatabaseService delegate, String userRole) {
        this.delegate = delegate;
        this.userRole = userRole;
    }

    @Override
    public String query(String sql) {
        // All roles can query
        System.out.println("[SEC] User role '" + userRole + "' querying...");
        return delegate.query(sql);
    }

    @Override
    public void execute(String sql) {
        // Only ADMIN can execute write operations
        if (!"ADMIN".equals(userRole)) {
            throw new SecurityException("Access denied: User role '" + userRole + "' cannot execute DDL/DML");
        }
        System.out.println("[SEC] ADMIN executing...");
        delegate.execute(sql);
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        // Caching proxy — virtual + caching
        DatabaseService cachingProxy = new CachingDatabaseProxy();
        System.out.println(cachingProxy.query("SELECT * FROM users")); // DB query
        System.out.println(cachingProxy.query("SELECT * FROM users")); // Cache hit!
        System.out.println(cachingProxy.query("SELECT * FROM orders")); // DB query

        System.out.println();

        // Protection proxy for admin
        DatabaseService adminProxy = new SecureDatabaseProxy(new RealDatabaseService(), "ADMIN");
        adminProxy.query("SELECT 1");
        adminProxy.execute("DELETE FROM temp");  // ✅ ADMIN allowed

        // Protection proxy for regular user
        DatabaseService userProxy = new SecureDatabaseProxy(new RealDatabaseService(), "USER");
        userProxy.query("SELECT name FROM products");  // ✅ OK
        try {
            userProxy.execute("DROP TABLE users");      // ❌ SecurityException
        } catch (SecurityException e) {
            System.out.println("🚫 " + e.getMessage());
        }
    }
}

JDK Dynamic Proxy — Spring's Core Mechanism

JDK Dynamic Proxy
import java.lang.reflect.*;

// JDK Dynamic Proxy — creates proxy at runtime without writing a class
DatabaseService loggingProxy = (DatabaseService) Proxy.newProxyInstance(
    DatabaseService.class.getClassLoader(),
    new Class[]{ DatabaseService.class },
    new InvocationHandler() {
        private final DatabaseService delegate = new RealDatabaseService();

        @Override
        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
            System.out.println("→ BEFORE: " + method.getName());
            Object result = method.invoke(delegate, args);
            System.out.println("← AFTER: " + method.getName());
            return result;
        }
    }
);

loggingProxy.query("SELECT * FROM products");
// → BEFORE: query
// ⚡ DB QUERY: SELECT * FROM products
// ← AFTER: query

// This is EXACTLY how Spring @Transactional, @Cacheable, @Async work!

🎤 Interview Q&A

When you annotate a method with @Transactional, Spring creates a proxy (JDK Dynamic Proxy if implements interface, CGLIB subclass proxy otherwise) around your bean. When a method is called, the proxy intercepts it, starts a transaction, calls the real method, then commits or rolls back based on result. Your business class knows nothing about transaction management — clean separation via Proxy/AOP.

JDK Dynamic Proxy: Creates proxy by implementing the target's interface at runtime. Requires that the target implements at least one interface. Uses java.lang.reflect.Proxy.
CGLIB: Creates proxy by subclassing the target class at runtime. Works with classes that don't implement interfaces. Cannot proxy final classes or final methods. Spring Boot prefers CGLIB by default since Spring 4.

All three wrap an object, but:
Proxy: Same interface, controls access (security, caching, lazy loading). Client doesn't know it's a proxy.
Decorator: Same interface, adds behavior/features. Client often composes decorators.
Adapter: Different interface, converts between incompatible interfaces. Structural compatibility.
