Bridge Pattern

Separates an object's abstraction from its implementation so that the two can vary independently. Prevents a "Cartesian product" class explosion.

Problem It Solves

Without Bridge: RedCircle, BlueCircle, RedSquare, BlueSquare... With Bridge: just Shape + Color — 2+2 = 4 classes instead of 2×2 = 4 and growing exponentially.

Java Implementation

Notification System with Bridge - Implementation side (HOW to deliver)
//  Implementation side (HOW to deliver) 
public interface MessageSender {
    void sendMessage(String recipient, String content);
}

public class EmailSender implements MessageSender {
    public void sendMessage(String recipient, String content) {
        System.out.println(" EMAIL → " + recipient + ": " + content);
    }
}

public class SMSSender implements MessageSender {
    public void sendMessage(String recipient, String content) {
        System.out.println(" SMS → " + recipient + ": " + content);
    }
}

public class SlackSender implements MessageSender {
    public void sendMessage(String recipient, String content) {
        System.out.println(" SLACK → @" + recipient + ": " + content);
    }
}
Notification System with Bridge - Abstraction side (WHAT type of message)
//  Abstraction side (WHAT type of message) 
public abstract class Notification {
    // Bridge — the implementation is injected
    protected final MessageSender sender;

    public Notification(MessageSender sender) {
        this.sender = sender;
    }

    public abstract void notify(String userId, String message);
}
Notification System with Bridge - Refined abstractions — message types
// Refined abstractions — message types
public class UrgentNotification extends Notification {
    public UrgentNotification(MessageSender sender) { super(sender); }

    @Override
    public void notify(String userId, String message) {
        sender.sendMessage(userId, " URGENT: " + message.toUpperCase());
    }
}

public class InfoNotification extends Notification {
    public InfoNotification(MessageSender sender) { super(sender); }

    @Override
    public void notify(String userId, String message) {
        sender.sendMessage(userId, " INFO: " + message);
    }
}

public class MarketingNotification extends Notification {
    public MarketingNotification(MessageSender sender) { super(sender); }

    @Override
    public void notify(String userId, String message) {
        sender.sendMessage(userId, " OFFER: " + message + " - Limited time!");
    }
}
Notification System with Bridge - Usage — Mix and match any abstraction × any implementation
// Usage — Mix and match any abstraction × any implementation
public class Main {
    public static void main(String[] args) {
        // Urgent via Email
        Notification urgentEmail = new UrgentNotification(new EmailSender());
        urgentEmail.notify("alice@example.com", "Server is down!");

        // Urgent via SMS
        Notification urgentSMS = new UrgentNotification(new SMSSender());
        urgentSMS.notify("+919999999999", "Server is down!");

        // Marketing via Slack
        Notification marketingSlack = new MarketingNotification(new SlackSender());
        marketingSlack.notify("channel-general", "New feature released!");

        // Info via Email
        Notification infoEmail = new InfoNotification(new EmailSender());
        infoEmail.notify("bob@example.com", "Your report is ready.");

        // Bridge: 3 abstractions × 3 implementations = 9 combinations with just 6 classes!
    }
}

Interview Q&A

Structurally similar, but intent differs. Strategy: Defines interchangeable behaviors/algorithms for a context. The context uses a strategy object. Focus: vary the algorithm.
Bridge: Decouples abstraction hierarchy from implementation hierarchy so both evolve independently. Has two full hierarchies (Abstraction + Implementation). Focus: separation of concerns in class structure.

Choose Bridge when you have two independent dimensions of variation (e.g., shape type AND color, or message type AND delivery channel). With inheritance, adding a new dimension multiplies your class count. Bridge keeps the dimensions orthogonal — adding a new sender doesn't require changes to any notification class.
