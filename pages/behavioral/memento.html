18. Memento Pattern

Captures and externalizes an object's internal state so that the object can be restored to that state later, without violating encapsulation.

Java Implementation

Game Checkpoint / Undo System
import java.util.ArrayDeque;
import java.util.Deque;
Game Checkpoint / Undo System - Memento — stores a snapshot of the Originator's state
// Memento — stores a snapshot of the Originator's state
Game Checkpoint / Undo System - Made as an inner record for encapsulation (Java 16+)
// Made as an inner record for encapsulation (Java 16+)
public class GameCharacter {

    //  Memento (inner) 
    public record Memento(int health, int level, String location, int gold) {}

    // Originator state
    private int health;
    private int level;
    private String location;
    private int gold;

    public GameCharacter(String startLocation) {
        this.health   = 100;
        this.level    = 1;
        this.location = startLocation;
        this.gold     = 0;
    }

    // Creates a memento (snapshot)
    public Memento save() {
        System.out.println(" CHECKPOINT saved at " + location
            + " | HP:" + health + " | LVL:" + level + " | Gold:" + gold);
        return new Memento(health, level, location, gold);
    }

    // Restores from memento
    public void restore(Memento m) {
        this.health   = m.health();
        this.level    = m.level();
        this.location = m.location();
        this.gold     = m.gold();
        System.out.println(" RESTORED to " + location
            + " | HP:" + health + " | LVL:" + level + " | Gold:" + gold);
    }

    // Game actions that mutate state
    public void takeDamage(int dmg) { health = Math.max(0, health - dmg); }
    public void gainGold(int amount) { gold += amount; }
    public void levelUp() { level++; health = 100; }
    public void moveTo(String loc) { this.location = loc; }
    public void printStatus() {
        System.out.printf("  Status: %s | HP:%d | LVL:%d | Gold:%d%n", location, health, level, gold);
    }
}
Game Checkpoint / Undo System - Caretaker — manages the history of mementos
// Caretaker — manages the history of mementos
public class GameSaveManager {
    private final Deque<GameCharacter.Memento> history = new ArrayDeque<>();

    public void saveCheckpoint(GameCharacter character) {
        history.push(character.save());
    }

    public void loadLastCheckpoint(GameCharacter character) {
        if (history.isEmpty()) {
            System.out.println("No checkpoint available!");
            return;
        }
        character.restore(history.pop());
    }

    public int checkpointCount() { return history.size(); }
}
Game Checkpoint / Undo System - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        GameCharacter hero = new GameCharacter("Village");
        GameSaveManager saveManager = new GameSaveManager();

        // Save at start
        saveManager.saveCheckpoint(hero);

        // Adventure begins
        hero.moveTo("Forest");
        hero.gainGold(50);
        hero.takeDamage(30);
        hero.printStatus(); // Forest | HP:70 | LVL:1 | Gold:50

        // Save progress
        saveManager.saveCheckpoint(hero);

        // Boss battle — things go wrong
        hero.moveTo("Dragon Cave");
        hero.takeDamage(90); // near death
        hero.printStatus(); // Dragon Cave | HP:0 | LVL:1 | Gold:50

        // Load last checkpoint!
        saveManager.loadLastCheckpoint(hero);
        hero.printStatus(); // Forest | HP:70 | LVL:1 | Gold:50 ← restored!
    }
}

Interview Q&A

The Memento stores state but the Caretaker (which holds Mementos) cannot access or modify the state directly — the Memento's fields are accessible only to the Originator. In Java, this is achieved via inner class (private access), package-private visibility, or sealed/record types. The Caretaker treats Memento as an opaque token.

(1) Serialization — serialize the object to bytes and deserialize to restore. (2) Prototype (clone) — take a deep copy of the object. (3) Event Sourcing — store the sequence of events that led to the current state; replay to restore. (4) Database snapshots — persist state to DB. Event Sourcing is the enterprise-scale Memento.
