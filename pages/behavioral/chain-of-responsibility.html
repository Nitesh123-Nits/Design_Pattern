13. Chain of Responsibility Pattern

Passes a request along a chain of handlers. Each handler decides either to process the request or to pass it to the next handler in the chain.

When to Use

  • When more than one object may handle a request and the handler isn't known a priori
  • When you want to issue a request to one of several objects without specifying the receiver explicitly
  • Middleware pipelines, filter chains, logging levels
  • Spring Security filter chain, Java Servlet filters

Java Implementation

ATM Dispenser Chain - Handler Interface
// Handler Interface
public abstract class DispenseChain {
    protected DispenseChain nextChain;

    public void setNext(DispenseChain next) {
        this.nextChain = next;
    }

    public abstract void dispense(Currency currency);
}

public class Currency {
    private final int amount;
    public Currency(int amount) { this.amount = amount; }
    public int getAmount() { return amount; }
}
ATM Dispenser Chain - Concrete Handlers
// Concrete Handlers
public class Rupee2000Handler extends DispenseChain {
    @Override
    public void dispense(Currency currency) {
        if (currency.getAmount() >= 2000) {
            int num = currency.getAmount() / 2000;
            int remainder = currency.getAmount() % 2000;
            System.out.println("Dispensing " + num + " Rs. 2000 note(s)");
            if (remainder != 0) nextChain.dispense(new Currency(remainder));
        } else {
            nextChain.dispense(currency);
        }
    }
}

    @Override
    public void dispense(Currency currency) {
        if (currency.getAmount() >= 100) {
            int num = currency.getAmount() / 100;
            int remainder = currency.getAmount() % 100;
            System.out.println("Dispensing " + num + " Rs. 100 note(s)");
            if (remainder != 0) {
                System.out.println("Remaining amount " + remainder + " cannot be dispensed.");
            }
        } else {
             System.out.println("Amount " + currency.getAmount() + " is too small to dispense.");
        }
    }
}
ATM Dispenser Chain - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        // Initialize the chain
        DispenseChain c1 = new Rupee2000Handler();
        DispenseChain c2 = new Rupee500Handler();
        DispenseChain c3 = new Rupee200Handler();
        DispenseChain c4 = new Rupee100Handler();

        // Set the sequence
        c1.setNext(c2);
        c2.setNext(c3);
        c3.setNext(c4);

        System.out.println("Processing Rs. 5800:");
        c1.dispense(new Currency(5800));

        System.out.println("\nProcessing Rs. 2350:");
        c1.dispense(new Currency(2350));
    }
}

Real-World Usage

  • Spring Security Filter Chain — each filter handles or passes
  • Java Servlet Filter Chain (FilterChain.doFilter())
  • Logging levels (DEBUG → INFO → WARN → ERROR)
  • Exception handling chains (try-catch hierarchy)

Interview Q&A

Spring Security has a SecurityFilterChain — an ordered list of Filters. Each filter (e.g., JwtAuthenticationFilter, UsernamePasswordAuthenticationFilter, ExceptionTranslationFilter) processes the request, optionally modifies the SecurityContext, then calls chain.doFilter() to pass to the next. The chain is configured with HttpSecurity in Spring Boot and is a textbook CoR implementation.

(1) No guarantee of handling — request might reach end without being processed. (2) Debugging is harder — hard to see which handler will respond without stepping through. (3) Performance — chain traversal adds overhead for each request. (4) Ordering matters — improper chain setup can lead to wrong handler handling a request.
