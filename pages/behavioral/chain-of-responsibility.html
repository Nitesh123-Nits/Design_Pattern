⛓️ Chain of Responsibility

Passes a request along a chain of handlers. Each handler decides either to process the request or to pass it to the next handler in the chain.

🎯 When to Use

  • When more than one object may handle a request and the handler isn't known a priori
  • When you want to issue a request to one of several objects without specifying the receiver explicitly
  • Middleware pipelines, filter chains, logging levels
  • Spring Security filter chain, Java Servlet filters

💻 Java Implementation

Purchase Approval Chain
// ── Handler Interface ─────────────────────────────────────────
public abstract class ApprovalHandler {
    protected ApprovalHandler nextHandler;

    public ApprovalHandler setNext(ApprovalHandler next) {
        this.nextHandler = next;
        return next; // enables chaining: handler1.setNext(handler2).setNext(handler3)
    }

    public abstract void handleRequest(PurchaseRequest request);

    protected void passToNext(PurchaseRequest request) {
        if (nextHandler != null) {
            nextHandler.handleRequest(request);
        } else {
            System.out.println("❌ Request denied: $" + request.getAmount() + " — no approver available");
        }
    }
}

public class PurchaseRequest {
    private final String purpose;
    private final double amount;
    public PurchaseRequest(String purpose, double amount) { this.purpose = purpose; this.amount = amount; }
    public String getPurpose() { return purpose; }
    public double getAmount() { return amount; }
}

// ── Concrete Handlers ─────────────────────────────────────────
public class TeamLeadHandler extends ApprovalHandler {
    private static final double LIMIT = 1000.0;

    @Override
    public void handleRequest(PurchaseRequest request) {
        if (request.getAmount() <= LIMIT) {
            System.out.printf("✅ TEAM LEAD approved $%.2f for '%s'%n",
                request.getAmount(), request.getPurpose());
        } else {
            System.out.printf("⏩ TEAM LEAD passing $%.2f to Manager%n", request.getAmount());
            passToNext(request);
        }
    }
}

public class ManagerHandler extends ApprovalHandler {
    private static final double LIMIT = 10000.0;

    @Override
    public void handleRequest(PurchaseRequest request) {
        if (request.getAmount() <= LIMIT) {
            System.out.printf("✅ MANAGER approved $%.2f for '%s'%n",
                request.getAmount(), request.getPurpose());
        } else {
            System.out.printf("⏩ MANAGER passing $%.2f to Director%n", request.getAmount());
            passToNext(request);
        }
    }
}

public class DirectorHandler extends ApprovalHandler {
    private static final double LIMIT = 100000.0;

    @Override
    public void handleRequest(PurchaseRequest request) {
        if (request.getAmount() <= LIMIT) {
            System.out.printf("✅ DIRECTOR approved $%.2f for '%s'%n",
                request.getAmount(), request.getPurpose());
        } else {
            System.out.printf("⏩ DIRECTOR passing $%.2f to CEO%n", request.getAmount());
            passToNext(request);
        }
    }
}

public class CEOHandler extends ApprovalHandler {
    @Override
    public void handleRequest(PurchaseRequest request) {
        // CEO approves everything (final handler)
        System.out.printf("✅ CEO approved $%.2f for '%s'%n",
            request.getAmount(), request.getPurpose());
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        // Build the chain
        ApprovalHandler teamLead = new TeamLeadHandler();
        ApprovalHandler manager  = new ManagerHandler();
        ApprovalHandler director = new DirectorHandler();
        ApprovalHandler ceo      = new CEOHandler();

        teamLead.setNext(manager).setNext(director).setNext(ceo);

        // Same entry point — each request finds its appropriate approver
        teamLead.handleRequest(new PurchaseRequest("Office supplies", 500));
        teamLead.handleRequest(new PurchaseRequest("Team training", 5000));
        teamLead.handleRequest(new PurchaseRequest("Software licenses", 50000));
        teamLead.handleRequest(new PurchaseRequest("Acquisition", 500000));
    }
}

🔍 Real-World Usage

  • Spring Security Filter Chain — each filter handles or passes
  • Java Servlet Filter Chain (FilterChain.doFilter())
  • Logging levels (DEBUG → INFO → WARN → ERROR)
  • Exception handling chains (try-catch hierarchy)

🎤 Interview Q&A

Spring Security has a SecurityFilterChain — an ordered list of Filters. Each filter (e.g., JwtAuthenticationFilter, UsernamePasswordAuthenticationFilter, ExceptionTranslationFilter) processes the request, optionally modifies the SecurityContext, then calls chain.doFilter() to pass to the next. The chain is configured with HttpSecurity in Spring Boot and is a textbook CoR implementation.

(1) No guarantee of handling — request might reach end without being processed. (2) Debugging is harder — hard to see which handler will respond without stepping through. (3) Performance — chain traversal adds overhead for each request. (4) Ordering matters — improper chain setup can lead to wrong handler handling a request.
