Command Pattern

Encapsulates a request as an object, allowing parameterization, queuing, logging, and undoable operations. Decouples the sender of a request from the object that handles it.

When to Use

  • Undo/Redo functionality (text editors, IDEs)
  • Queuing and scheduling requests (job queues, task schedulers)
  • Transactional behavior — commit/rollback batch of operations
  • Macro recording — record then replay sequences of actions

Java Implementation

Text Editor with Undo/Redo
import java.util.ArrayDeque;
import java.util.Deque;
Text Editor with Undo/Redo - Command Interface
//  Command Interface 
public interface Command {
    void execute();
    void undo();
}
Text Editor with Undo/Redo - Receiver — the object that actually does the work
//  Receiver — the object that actually does the work 
public class TextDocument {
    private StringBuilder content = new StringBuilder();

    public void insertText(int position, String text) {
        content.insert(position, text);
        System.out.println("Document: \"" + content + "\"");
    }

    public void deleteText(int position, int length) {
        content.delete(position, position + length);
        System.out.println("Document: \"" + content + "\"");
    }

    public String getContent() { return content.toString(); }
    public int length() { return content.length(); }
}
Text Editor with Undo/Redo - Concrete Commands
//  Concrete Commands 
public class InsertTextCommand implements Command {
    private final TextDocument doc;
    private final int position;
    private final String text;

    public InsertTextCommand(TextDocument doc, int position, String text) {
        this.doc = doc;
        this.position = position;
        this.text = text;
    }

    @Override
    public void execute() {
        doc.insertText(position, text);
    }

    @Override
    public void undo() {
        // Reverse: delete what we inserted
        doc.deleteText(position, text.length());
    }
}

public class DeleteTextCommand implements Command {
    private final TextDocument doc;
    private final int position;
    private final int length;
    private String deletedText; // saved for undo

    public DeleteTextCommand(TextDocument doc, int position, int length) {
        this.doc = doc;
        this.position = position;
        this.length = length;
    }

    @Override
    public void execute() {
        // Save before deleting (for undo)
        deletedText = doc.getContent().substring(position, position + length);
        doc.deleteText(position, length);
    }

    @Override
    public void undo() {
        // Restore the deleted text
        doc.insertText(position, deletedText);
    }
}
Text Editor with Undo/Redo - Invoker — manages command execution and history
//  Invoker — manages command execution and history 
public class TextEditor {
    private final TextDocument doc = new TextDocument();
    private final Deque<Command> undoStack = new ArrayDeque<>();
    private final Deque<Command> redoStack = new ArrayDeque<>();

    public void executeCommand(Command cmd) {
        cmd.execute();
        undoStack.push(cmd);
        redoStack.clear(); // new action clears redo history
    }

    public void undo() {
        if (undoStack.isEmpty()) { System.out.println("Nothing to undo"); return; }
        Command cmd = undoStack.pop();
        cmd.undo();
        redoStack.push(cmd);
        System.out.println("[UNDO performed]");
    }

    public void redo() {
        if (redoStack.isEmpty()) { System.out.println("Nothing to redo"); return; }
        Command cmd = redoStack.pop();
        cmd.execute();
        undoStack.push(cmd);
        System.out.println("[REDO performed]");
    }

    public TextDocument getDocument() { return doc; }
}
Text Editor with Undo/Redo - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();

        editor.executeCommand(new InsertTextCommand(editor.getDocument(), 0, "Hello"));
        editor.executeCommand(new InsertTextCommand(editor.getDocument(), 5, " World"));
        editor.executeCommand(new DeleteTextCommand(editor.getDocument(), 5, 6)); // delete " World"

        editor.undo(); // restore " World"
        editor.undo(); // remove " World" again
        editor.redo(); // re-insert " World"
    }
}

Interview Q&A

Command: Interface with execute() and optionally undo().
ConcreteCommand: Binds an action to a Receiver — knows how to execute and undo.
Receiver: The actual object that knows how to perform the operation.
Invoker: Asks the command to carry out a request; may maintain a history for undo/redo.

Runnable is a simplified Command pattern — run() is the execute() method. ExecutorService is the Invoker. ThreadPoolExecutor queues and executes Runnables. Java 8's Callable extends this with a return value. Lambdas make defining commands extremely concise: executor.submit(() -> doWork()).

Create a BatchCommand (Composite Command) that holds a list of commands. In execute(): execute all commands sequentially. On failure: call undo() on all previously executed commands in reverse order (simulating rollback). This is essentially how database transactions work conceptually.
