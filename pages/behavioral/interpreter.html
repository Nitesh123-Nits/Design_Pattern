15. Interpreter Pattern

Given a language, defines a representation for its grammar along with an interpreter that uses the representation to interpret sentences in the language.

When to Use

  • When you have a domain-specific language (DSL) to interpret
  • Simple expression evaluators (math, boolean logic)
  • Spring Expression Language (SpEL) — #{1 + 2}
  • SQL parsers, regular expression engines, scripting engines

Java Implementation

Boolean Expression Interpreter
import java.util.Map;
Boolean Expression Interpreter - Abstract Expression
// Abstract Expression
public interface BooleanExpression {
    boolean interpret(Map<String, Boolean> context);
}
Boolean Expression Interpreter - Terminal expressions (leaves)
// Terminal expressions (leaves)
public class VariableExpression implements BooleanExpression {
    private final String name;
    public VariableExpression(String name) { this.name = name; }

    @Override
    public boolean interpret(Map<String, Boolean> context) {
        return context.getOrDefault(name, false);
    }
}

public class LiteralExpression implements BooleanExpression {
    private final boolean value;
    public LiteralExpression(boolean value) { this.value = value; }

    @Override
    public boolean interpret(Map<String, Boolean> context) { return value; }
}
Boolean Expression Interpreter - Non-terminal (composite) expressions
// Non-terminal (composite) expressions
public class AndExpression implements BooleanExpression {
    private final BooleanExpression left, right;
    public AndExpression(BooleanExpression l, BooleanExpression r) { left=l; right=r; }

    @Override
    public boolean interpret(Map<String, Boolean> ctx) {
        return left.interpret(ctx) && right.interpret(ctx);
    }
}

public class OrExpression implements BooleanExpression {
    private final BooleanExpression left, right;
    public OrExpression(BooleanExpression l, BooleanExpression r) { left=l; right=r; }

    @Override
    public boolean interpret(Map<String, Boolean> ctx) {
        return left.interpret(ctx) || right.interpret(ctx);
    }
}

public class NotExpression implements BooleanExpression {
    private final BooleanExpression expr;
    public NotExpression(BooleanExpression expr) { this.expr = expr; }

    @Override
    public boolean interpret(Map<String, Boolean> ctx) {
        return !expr.interpret(ctx);
    }
}
Boolean Expression Interpreter - Usage — build and evaluate: (isAdmin OR isManager) AND NOT isLocked
// Usage — build and evaluate: (isAdmin OR isManager) AND NOT isLocked
public class Main {
    public static void main(String[] args) {
        // Build expression tree
        BooleanExpression isAdmin   = new VariableExpression("isAdmin");
        BooleanExpression isManager = new VariableExpression("isManager");
        BooleanExpression isLocked  = new VariableExpression("isLocked");

        BooleanExpression canAccess = new AndExpression(
            new OrExpression(isAdmin, isManager),
            new NotExpression(isLocked)
        );

        // Context 1: admin, not locked
        Map<String, Boolean> adminCtx = Map.of("isAdmin", true, "isManager", false, "isLocked", false);
        System.out.println("Admin access: " + canAccess.interpret(adminCtx)); // true

        // Context 2: manager, locked
        Map<String, Boolean> lockedMgr = Map.of("isAdmin", false, "isManager", true, "isLocked", true);
        System.out.println("Locked manager: " + canAccess.interpret(lockedMgr)); // false

        // Context 3: regular user
        Map<String, Boolean> userCtx = Map.of("isAdmin", false, "isManager", false, "isLocked", false);
        System.out.println("Regular user: " + canAccess.interpret(userCtx)); // false
    }
}

Interview Q&A

(1) Complex grammars lead to many classes (one per grammar rule). (2) Hard to maintain as grammar evolves. (3) Poor performance for complex expressions. For complex language parsing, use parser generators (ANTLR), PEG parsers, or established DSL frameworks. Interpreter is best for simple, small grammars.

Spring Expression Language (SpEL) is a full Interpreter implementation. ExpressionParser parser = new SpelExpressionParser(); parses and evaluates expressions like "#{user.age > 18 && user.role == 'ADMIN'}". The parser builds an AST (Abstract Syntax Tree) from the grammar, then the interpreter evaluates it against a context — classic Interpreter pattern at enterprise scale.
