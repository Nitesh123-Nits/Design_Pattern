16. Iterator Pattern

Provides a way to access elements of a collection object sequentially without exposing its underlying representation. Java's Iterator and for-each are built on this pattern.

Java Implementation

Custom Collection with Iterator
import java.util.Iterator;
import java.util.NoSuchElementException;
Custom Collection with Iterator - Custom collection — implements Iterable to support for-each
// Custom collection — implements Iterable to support for-each
public class NumberRange implements Iterable<Integer> {
    private final int start;
    private final int end;
    private final int step;

    public NumberRange(int start, int end, int step) {
        this.start = start;
        this.end   = end;
        this.step  = step;
    }

    @Override
    public Iterator<Integer> iterator() {
        return new RangeIterator();
    }

    // Inner iterator — knows collection internals, client doesn't need to
    private class RangeIterator implements Iterator<Integer> {
        private int current = start;

        @Override
        public boolean hasNext() { return current <= end; }

        @Override
        public Integer next() {
            if (!hasNext()) throw new NoSuchElementException();
            int value = current;
            current += step;
            return value;
        }
    }
}
Custom Collection with Iterator - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        NumberRange evens = new NumberRange(0, 10, 2);

        // for-each works because we implement Iterable
        for (int n : evens) {
            System.out.print(n + " "); // 0 2 4 6 8 10
        }

        // Manual iteration
        Iterator<Integer> it = new NumberRange(1, 5, 1).iterator();
        while (it.hasNext()) {
            System.out.print(it.next() + " "); // 1 2 3 4 5
        }

        // Java built-in Iterator example
        java.util.List<String> list = java.util.List.of("Java", "Spring", "Patterns");
        Iterator<String> strIt = list.iterator();
        while (strIt.hasNext()) {
            System.out.println(strIt.next());
        }
    }
}

Interview Q&A

Iterator: traverse forward only, has hasNext(), next(), remove(). Works on any Collection.
ListIterator: extends Iterator, supports bidirectional traversal (hasPrevious(), previous()), add(), set() — can modify during iteration. Only for List implementations.

Thrown when a collection is structurally modified while iterating over it (except through the iterator's own remove()). Fix: (1) Use iterator.remove() instead of list.remove() during iteration. (2) Use CopyOnWriteArrayList for concurrent scenario. (3) Collect items to remove, then remove after loop. (4) Use list.removeIf(predicate) (Java 8+).
