22. Template Method Pattern

Defines the skeleton of an algorithm in a base class, deferring some steps to subclasses. Lets subclasses redefine certain steps without changing the algorithm's overall structure.

When to Use

  • When multiple classes share the same algorithm structure but differ in specific steps
  • When you want to control which steps subclasses are allowed to override
  • Spring's JdbcTemplate, RestTemplate, AbstractController
  • Report generation, data exporters with common headers/footers

Java Implementation

Data Report Generator - Abstract class with the template method
// Abstract class with the template method
public abstract class ReportGenerator {

    //  Template Method — the skeleton. FINAL so no subclass can break the algorithm 
    public final void generateReport(List<String> data) {
        printHeader();          // step 1: always same structure
        processData(data);      // step 2: abstract — subclass provides
        applyFormatting();      // step 3: abstract — subclass provides
        printFooter();          // step 4: always same structure
        if (shouldAddSignature()) {  // hook — optional step, default: false
            addSignature();
        }
    }

    // Concrete methods — same for all subclasses
    private void printHeader() {
        System.out.println("==============================");
        System.out.println("    " + getReportTitle());
        System.out.println("    Generated: " + java.time.LocalDate.now());
        System.out.println("==============================");
    }

    private void printFooter() {
        System.out.println("==============================");
        System.out.println("End of Report");
        System.out.println("==============================");
    }

    // Abstract methods — subclasses MUST implement these
    protected abstract String getReportTitle();
    protected abstract void processData(List<String> data);
    protected abstract void applyFormatting();

    // Hooks — optional overrides with default behavior
    protected boolean shouldAddSignature() { return false; }
    protected void addSignature() { System.out.println("Signed off."); }
}
Data Report Generator - Concrete subclass 1 — CSV report
// Concrete subclass 1 — CSV report
public class CSVReportGenerator extends ReportGenerator {
    @Override
    protected String getReportTitle() { return "CSV Sales Report"; }

    @Override
    protected void processData(List<String> data) {
        System.out.println("Processing " + data.size() + " records into CSV format...");
        data.forEach(row -> System.out.println(row + ","));
    }

    @Override
    protected void applyFormatting() {
        System.out.println("Applying CSV formatting: comma-separated, UTF-8 encoded");
    }
}
Data Report Generator - Concrete subclass 2 — HTML report (with signature hook)
// Concrete subclass 2 — HTML report (with signature hook)
public class HTMLReportGenerator extends ReportGenerator {
    @Override
    protected String getReportTitle() { return "HTML Sales Report"; }

    @Override
    protected void processData(List<String> data) {
        System.out.println("<table>");
        data.forEach(row -> System.out.println("  <tr><td>" + row + "</td></tr>"));
        System.out.println("</table>");
    }

    @Override
    protected void applyFormatting() {
        System.out.println("Applying Bootstrap CSS table classes...");
    }

    // Override hook to add signature
    @Override
    protected boolean shouldAddSignature() { return true; }

    @Override
    protected void addSignature() {
        System.out.println("<footer>Report by Finance Team</footer>");
    }
}
Data Report Generator - Usage
// Usage
public class Main {
    public static void main(String[] args) {
        List<String> salesData = List.of("Product A: $1000", "Product B: $2500", "Product C: $750");

        System.out.println("--- CSV Report ---");
        new CSVReportGenerator().generateReport(salesData);

        System.out.println("\n--- HTML Report ---");
        new HTMLReportGenerator().generateReport(salesData);
    }
}

Real-World Usage

  • java.io.InputStream — read(byte[]) is implemented using abstract read()
  • Spring's JdbcTemplate — template method for query lifecycle
  • Spring's AbstractController — handleRequestInternal()
  • JUnit's test lifecycle (@BeforeEach, test methods, @AfterEach)

Interview Q&A

Template Method: Uses inheritance. Algorithm skeleton in base class; subclasses fill in steps. The variation points are defined at design time. Less flexible — must subclass to change behavior.
Strategy: Uses composition. The complete algorithm is encapsulated in a strategy object and can be swapped at runtime. Prefer Strategy over Template Method when you want runtime flexibility (favors composition over inheritance principle).

A hook is a method in the base class with a default (usually empty or no-op) implementation that subclasses may override — it's optional. Hooks allow subclasses to plug into the algorithm at specific points without making override mandatory. The shouldAddSignature() returning false by default is a boolean hook — the HTML subclass opts in by returning true.
