Prototype Pattern

Creates new objects by cloning an existing prototype object. Useful when object creation is expensive and you want copies with slight modifications.

When to Use

  • When creating an object is expensive (DB load, API call, complex init)
  • When you need many similar objects with minor differences
  • When you want to hide the complexity of creating new instances from the client
  • Game development: clone enemy/character templates

Java Implementation

Prototype — Shallow vs Deep Copy
import java.util.ArrayList;
import java.util.List;

// Prototype interface
public interface Prototype<T> {
    T clone();
}

//  Shallow Copy (default Java clone) 
public class Address {
    public String city;
    public String street;
    public Address(String city, String street) { this.city = city; this.street = street; }
}

//  Deep Copy Example 
public class Employee implements Prototype<Employee> {
    private String name;
    private String role;
    private List<String> skills;   // mutable collection — deep copy needed
    private Address address;        // mutable object — deep copy needed

    public Employee(String name, String role, List<String> skills, Address address) {
        this.name    = name;
        this.role    = role;
        this.skills  = new ArrayList<>(skills);  // defensive copy
        this.address = address;
    }

    // Deep clone — copy collections and mutable objects
    @Override
    public Employee clone() {
        return new Employee(
            this.name,
            this.role,
            new ArrayList<>(this.skills),          // deep copy the list
            new Address(this.address.city, this.address.street) // deep copy address
        );
    }

    public void addSkill(String skill) { skills.add(skill); }
    public void setRole(String role)   { this.role = role; }
    public void setCity(String city)   { this.address.city = city; }

    @Override
    public String toString() {
        return name + " | " + role + " | " + skills + " | " + address.city;
    }
}

//  Prototype Registry 
public class EmployeeRegistry {
    private static final Map<String, Employee> templates = new HashMap<>();

    static {
        // Pre-built templates — expensive to create, clone cheaply
        templates.put("JUNIOR_DEV", new Employee("", "Junior Developer",
            List.of("Java", "Git"), new Address("Remote", "N/A")));

        templates.put("SENIOR_DEV", new Employee("", "Senior Developer",
            List.of("Java", "Spring", "Kubernetes", "System Design"),
            new Address("Remote", "N/A")));

        templates.put("ARCHITECT", new Employee("", "Solution Architect",
            List.of("Java", "AWS", "Design Patterns", "Microservices"),
            new Address("HQ", "Tech Park")));
    }

    public static Employee create(String role, String name) {
        Employee template = templates.get(role);
        if (template == null) throw new IllegalArgumentException("Unknown role: " + role);
        Employee copy = template.clone(); // Clone the template
        // Customize the clone
        return copy;
    }
}

// Usage
public class Main {
    public static void main(String[] args) {
        Employee dev1 = EmployeeRegistry.create("SENIOR_DEV", "Alice");
        Employee dev2 = EmployeeRegistry.create("SENIOR_DEV", "Bob");

        dev1.addSkill("React");       // Only affects dev1
        dev1.setCity("New York");     // Deep copy — doesn't affect dev2

        System.out.println(dev1);     // Senior Developer | [Java, Spring, K8s, SD, React] | New York
        System.out.println(dev2);     // Senior Developer | [Java, Spring, K8s, SD] | Remote
    }
}

Interview Q&A

Shallow Copy: Copies primitive fields by value, but object references are shared. If you modify a list in the copy, the original is also affected. Java's default Object.clone() does a shallow copy.
Deep Copy: Recursively copies all nested objects so the clone is completely independent. Must be implemented manually (copy constructors, serialization, or manual field copying).

(1) Cloneable has no clone() method — it's just a marker interface. (2) Object.clone() is protected, requiring you to override and make it public. (3) The contract is fragile and poorly specified. (4) It does shallow copy by default, which is unsafe for mutable fields.
Alternatives: Copy constructors (new Employee(other)) or copy factory methods are preferred by Joshua Bloch (Effective Java).

(1) Copy constructor — manually copy all fields: new Employee(other). (2) Serialization — serialize to bytes then deserialize (works automatically for deep copy if all classes are Serializable, but slow). (3) Libraries — Apache Commons' SerializationUtils.clone(), or Kryo for fast serialization. (4) Mapping libraries — MapStruct or ModelMapper.
