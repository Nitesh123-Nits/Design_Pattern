Builder Pattern

Constructs a complex object step by step. Separates the construction of a complex object from its representation, allowing the same construction process to create different representations.

When to Use

  • When an object has many optional parameters (Telescoping Constructor problem)
  • When construction must be done step-by-step
  • When you want immutable objects with many fields
  • When you need to vary object representation (different configurations)

Java Implementation

Classic Builder (Inner Class) — Preferred in Java

HttpRequest.java
// Complex object with many optional parameters
public final class HttpRequest {

    // Required fields
    private final String url;
    private final String method;

    // Optional fields — would be a telescoping constructor nightmare
    private final Map<String, String> headers;
    private final String body;
    private final int timeoutMs;
    private final boolean followRedirects;
    private final String contentType;

    // Private constructor — only Builder can call this
    private HttpRequest(Builder builder) {
        this.url             = builder.url;
        this.method          = builder.method;
        this.headers         = Collections.unmodifiableMap(builder.headers);
        this.body            = builder.body;
        this.timeoutMs       = builder.timeoutMs;
        this.followRedirects = builder.followRedirects;
        this.contentType     = builder.contentType;
    }

    // Getters only — immutable object
    public String getUrl()          { return url; }
    public String getMethod()       { return method; }
    public String getBody()         { return body; }
    public int getTimeoutMs()       { return timeoutMs; }
    public boolean isFollowRedirects() { return followRedirects; }
    public Map<String, String> getHeaders() { return headers; }

    @Override
    public String toString() {
        return method + " " + url + " (timeout=" + timeoutMs + "ms, redirects=" + followRedirects + ")";
    }

    //  Static Inner Builder 
    public static class Builder {
        // Required
        private final String url;
        private final String method;

        // Optional — defaults set here
        private Map<String, String> headers = new HashMap<>();
        private String body = null;
        private int timeoutMs = 5000;
        private boolean followRedirects = true;
        private String contentType = "application/json";

        // Constructor for required parameters only
        public Builder(String method, String url) {
            if (url == null || url.isBlank()) throw new IllegalArgumentException("URL required");
            this.method = method.toUpperCase();
            this.url    = url;
        }

        // Each setter returns 'this' for method chaining
        public Builder header(String key, String value) {
            this.headers.put(key, value);
            return this;
        }

        public Builder body(String body) {
            this.body = body;
            return this;
        }

        public Builder timeout(int ms) {
            if (ms <= 0) throw new IllegalArgumentException("Timeout must be positive");
            this.timeoutMs = ms;
            return this;
        }

        public Builder followRedirects(boolean follow) {
            this.followRedirects = follow;
            return this;
        }

        public Builder contentType(String contentType) {
            this.contentType = contentType;
            return this;
        }

        // Terminal method — creates the immutable object
        public HttpRequest build() {
            // Final validation before building
            if ("POST".equals(method) || "PUT".equals(method)) {
                if (body == null) throw new IllegalStateException("Body required for " + method);
            }
            return new HttpRequest(this);
        }
    }
}

// Usage — Fluent, readable API
public class Main {
    public static void main(String[] args) {
        // GET request — minimal configuration
        HttpRequest getRequest = new HttpRequest.Builder("GET", "https://api.example.com/users")
                .header("Authorization", "Bearer eyJhbGci...")
                .header("Accept", "application/json")
                .timeout(3000)
                .build();

        // POST request — full configuration
        HttpRequest postRequest = new HttpRequest.Builder("POST", "https://api.example.com/users")
                .header("Authorization", "Bearer eyJhbGci...")
                .header("X-Request-ID", "req-123")
                .body("{\"name\":\"Alice\",\"email\":\"alice@example.com\"}")
                .contentType("application/json")
                .timeout(10000)
                .followRedirects(false)
                .build();

        System.out.println(getRequest);
        System.out.println(postRequest);
    }
}

Lombok @Builder — Industry Shortcut

User.java with Lombok
import lombok.Builder;
import lombok.Value;

@Value  // immutable — all fields final, no setters
@Builder(toBuilder = true)  // generates Builder class automatically
public class User {
    String id;
    String name;
    String email;
    String role;
    boolean active;

    // Usage
    public static void main(String[] args) {
        User user = User.builder()
                .id("usr-001")
                .name("Alice")
                .email("alice@example.com")
                .role("ADMIN")
                .active(true)
                .build();

        // toBuilder: copy + modify
        User updated = user.toBuilder()
                .role("USER")
                .build();
    }
}

Pros & Cons

AspectDetail
ReadableFluent API reads almost like English
ImmutabilityFinal fields set only via builder
ValidationValidate at build time before object creation
Optional paramsNo need for multiple constructors (telescoping)
VerbosityLots of boilerplate (use Lombok to mitigate)
Incomplete buildsDeveloper might forget to set required fields

Real-World Usage

  • StringBuilder — classic Java Builder for string construction
  • java.util.stream.Stream.Builder
  • HttpClient.newBuilder() (Java 11+)
  • Spring's UriComponentsBuilder, MockMvcRequestBuilders
  • Lombok's @Builder — industry standard for DTOs / entities

Interview Q&A

The Telescoping Constructor anti-pattern: when a class has many optional parameters, you end up with multiple overloaded constructors (or one with many nulls). This is error-prone — parameters of the same type can be accidentally swapped. Builder solves this with named setters for each field, validation before construction, and immutability of the final object.

Factory: Creates objects in one step, returning a product. Focuses on which type to create.
Builder: Creates complex objects step by step. Focuses on how to assemble a single complex object. Factory is about polymorphism; Builder is about construction.

(1) Put mandatory params in the Builder's constructor: new Builder("GET", "https://..."). (2) Throw exceptions in build() if required fields are null. (3) Use Step Builder pattern — method chain forces each required step before allowing the next (advanced). Lombok's @Builder supports @NonNull annotation to throw NPE for null required fields.
